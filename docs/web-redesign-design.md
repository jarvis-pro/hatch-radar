# Hatch Radar · Web 端重设计方案

> 立场：以产品经理 + 产品设计师视角，先诊断"为什么现在感觉劣质/单调"，再给出一套可落地的重设计。
> 范围：仅 `apps/web`（Vite + React Router SPA），不动后端契约、不动 mobile。
> 约束：继续用 shadcn/ui + Tailwind v4（CSS-first `@theme`），所以本方案绝大部分改动收敛在**设计令牌 + 应用外壳 + 少量组件**，工程风险可控。

---

## 0. 一句话主张

> **把"一个能用的后台模板"重做成"一台市场情报指挥台"。**

产品叫 Radar，本质是"把社区噪音炼成产品信号"的情报流水线（原始 → 闭环 → 漏斗）。但现在的 web 端没有把这个**情报 / 指挥台**的气质表达出来——它看起来像任何一个 shadcn 脚手架。重设计的目标不是"换皮肤变好看"，而是让界面的**空间结构、视觉语言、实时感**都在说同一句话：_这是一个严肃的、活着的、专业的情报系统_。

---

## 1. 现状诊断：劣质感从哪来

我不认为问题在"配色不够漂亮"。问题是三个**结构性缺失**，外加一层**精致度欠债**。

### 1.1 结构缺失①：没有"应用外壳"（最致命）

当前所有页面都活在一个 `max-w-5xl`（1024px）居中列里，顶部一条细导航 + 几个文字链接（[layout.tsx:35](apps/web/src/layout.tsx#L35)、[site-nav.tsx:8](apps/web/src/components/site-nav.tsx#L8)）。

这是**博客/落地页**的布局，不是**应用**的布局。后果：

- 宽屏上两侧大片留白，数据表格被挤在中间窄列里，看板信息憋屈；
- 导航是纯文字、**无图标**、无分组，扫一眼建立不起"我在一个系统里"的空间感；
- 没有持久的左侧栏 = 没有"系统骨架"。成熟工具（Linear / Vercel / Datadog / Sentry / Notion）无一例外是 **持久侧边栏 + 全幅工作区**。

> 这一条是"不像成熟操作系统"的**第一主因**。

### 1.2 结构缺失②：零品牌识别，纯 shadcn 默认灰

[globals.css:18](packages/ui/src/styles/globals.css#L18) 里 `--primary` 是接近黑的灰，所有中性色 chroma=0（`oklch(x 0 0)`），全站没有一抹品牌色。这就是 shadcn 开箱即用的样子——**它看起来像还没设计**。

- 用户在界面里找不到任何视觉锚点，所有元素一样重；
- "Radar / 信号 / 情报"这个天然富矿（雷达扫掠、信号强度、声呐）完全没被用上；
- 最关键的语义——痛点强度 HIGH/MEDIUM/LOW——在列表里只是灰色 badge，**最该被一眼看到的信号反而最沉默**。

> 这一条是"单调"的**第一主因**。

### 1.3 结构缺失③：层级太平，一切同权重

- 页面 h1 全是 `text-lg`（**18px**）+ 一句灰色 inline 副标题（[dashboard.tsx](apps/web/src/dashboard.tsx)、[insights.tsx](apps/web/src/pages/insights.tsx) 等）。18px 当一级标题，没有任何"重量"，进页面没有焦点。
- 所有卡片长一个样：`border + shadow-sm`。没有"海拔"体系，分不清主次面。
- 看板的图表是手搓的 `div` 柱子、32px 高（[dashboard.tsx:71](apps/web/src/dashboard.tsx#L71)）——一个号称"运行概览"的页面，可视化却很将就。

### 1.4 精致度欠债（次要但累积成"廉价"）

- **四种列表范式打架**：洞察是卡片、帖子是 `Item` 行、队列是 `Table`、分析是 `Table`。同一个系统四种列表语感。
- **筛选栏像补丁**：换行 flex 一排塞 select + 搜索 + 按钮（[filter-bar.tsx:49](apps/web/src/components/filter-bar.tsx#L49)），没有"工具栏"的体面。
- **空/载/错状态朴素**：骨架是灰块、空态是居中图标，没有动效、没有过渡，缺"打磨感"。
- **没有全局检索 / 命令面板 / 键盘流**——成熟工具的标配，恰恰是"专业感"最浓的部分。

## 2. 设计原则（每个决定都回到这五条）

1. **像系统，不像页面**：持久外壳 + 全幅工作区，进任何页面都"在系统里"。
2. **信号优先**：界面的颜色预算优先花在"内容语义"（强度/状态/趋势）上，而不是装饰。强度该红就红。
3. **活着**：这是一个每 30 分钟抓取、有队列、有 worker 的实时工厂——界面要有脉搏（实时状态、乐观更新、轻动效）。
4. **密度有纪律**：数据页该密则密（表格、紧凑行），阅读页该松则松（详情、洞察）。一套统一的密度语言，而不是四种。
5. **克制的品牌**：一个主品牌色 + 一套语义色，雷达隐喻只在关键时刻点睛（登录、实时状态、空态），不滥用。

## 3. 设计语言（视觉系统）

给这套系统起个内部代号：**"Signal"**——信号情报控制台。下面是可直接落进 `@theme` 的令牌建议。

### 3.1 色彩

三层颜色，**职责分明、互不串台**：

**A. 品牌色（导航高亮、主按钮、链接动作、聚焦环）**
选靛蓝偏紫（indigo-violet）。理由：① 专业现代 SaaS 的安全选择（Linear 气质）；② **刻意避开红/琥珀/绿**——那三色要留给"强度语义"，品牌色绝不能和内容语义抢色。

```css
--primary: oklch(0.55 0.21 277); /* 靛紫 */
--primary-foreground: oklch(0.98 0.01 277);
--signal: oklch(0.72 0.14 215); /* 青色，专用于"实时/在线/扫掠"等"活着"的状态 */
```

**B. 语义强度色（内容！痛点强度、洞察热度——全站统一）**
现在 `PAIN_ACCENT` 已经有 HIGH=红/MED=琥珀/LOW=翠绿（[insight-detail.tsx](apps/web/src/pages/insight-detail.tsx)），把它**升格为正式 token**，并贯穿列表/详情/看板：

```css
--intensity-high: oklch(0.62 0.22 25); /* 红：高强度痛点/强信号 */
--intensity-medium: oklch(0.76 0.16 70); /* 琥珀 */
--intensity-low: oklch(0.72 0.13 165); /* 翠绿 */
```

**C. 中性色——关键技巧：给灰"上色"**
当前中性色全是 `chroma 0` 的纯灰，这正是"未经设计"的廉价感来源。把所有中性色的 chroma 从 0 推到 `0.008~0.016`、色相对齐品牌 hue（277），灰就有了"被设计过"的高级感（冷调、成体系），肉眼几乎看不出偏色，但整体气质立变。

暗色（建议 **dark-first**，指挥台在暗色下最出彩）：

```css
--background: oklch(0.17 0.012 277); /* 近黑，冷调 */
--card: oklch(0.21 0.014 277); /* 抬升面 */
--popover: oklch(0.24 0.016 277);
--muted: oklch(0.25 0.012 277);
--border: oklch(1 0 0 / 8%);
--muted-foreground: oklch(0.7 0.02 277);
--foreground: oklch(0.96 0.005 277);
```

亮色：

```css
--background: oklch(0.99 0.004 277);
--card: oklch(1 0 0);
--muted: oklch(0.97 0.006 277);
--border: oklch(0.91 0.008 277);
--foreground: oklch(0.21 0.015 277);
```

> 落地成本极低：这些全是 `globals.css` 里已有变量的**改值**，组件零改动即可全站换气质。这是性价比最高的一步。

### 3.2 海拔（Elevation）—— 替代"全员 border+shadow-sm"

定义 3 级面，用"表面亮度 + 阴影"双通道表达层次，而非一刀切描边：

| 级别       | 用途                        | 暗色表面       | 阴影              |
| ---------- | --------------------------- | -------------- | ----------------- |
| L0 base    | 页面底                      | `--background` | 无                |
| L1 raised  | 卡片、侧栏、列表项          | `--card`       | `0 1px 2px /20%`  |
| L2 overlay | 弹窗、下拉、命令面板、Sheet | `--popover`    | `0 8px 24px /35%` |

主面板用"更亮一档的表面"自然浮起，**减少描边**（现在到处是 border，视觉很碎）。

### 3.3 排版

当前是系统字体栈 + h1 仅 18px，是"平"的另一主因。建议：

- **字体**：UI 用 **Inter**（或 Geist Sans）；数字/ID/Token/成本用 **JetBrains Mono**（或 Geist Mono）。"数据工厂"就该有等宽数字的工程气质（现在已用 `tabular-nums`，再上 mono 字形更对味）。
- **字阶**（建立真实层级，重点是把页面标题"做大做重"）：

| 角色          | 字号/行高 | 字重/字距           | 用在                        |
| ------------- | --------- | ------------------- | --------------------------- |
| Display       | 30/38     | 700 / -0.02em       | 登录、空系统引导            |
| **Page H1**   | **24/32** | 650 / -0.01em       | **页面标题（现在才 18px）** |
| Section H2    | 18/28     | 600                 | 卡片/分区标题               |
| H3            | 15/24     | 600                 | 子区块                      |
| Body          | 14/20     | 400                 | 正文/表格                   |
| Small         | 13/18     | 400                 | 辅助                        |
| Caption/Label | 12/16     | 500 / 0.04em / 大写 | 字段标签、KPI 标签          |
| Mono          | 13        | — / tabular         | 数值、ID、成本、Token       |

副标题不要再 inline 灰字跟在 h1 后面，改成 h1 下方独立一行 `text-muted-foreground`，标题区才立得住。

### 3.4 形状、间距、动效

- **圆角**：保留 `--radius: 0.625rem`（10px），统一即可；输入/按钮可略收到 8px 显"工程感"。
- **间距节奏**：沿用 Tailwind 8px 体系，但**页面级**用更大的呼吸（区块间 `gap-6/gap-8`，现在普遍 `gap-3/gap-4` 偏挤）。
- **动效原则**：①一切状态切换有过渡（120–200ms，`ease-out`）；②实时数据用"脉冲/微光"而非生硬刷新；③列表进出、弹窗用轻微位移+淡入；④**雷达隐喻只在登录页和顶栏实时指示器上点睛**（一圈缓慢扫掠），别处克制。

### 3.5 图标与品牌物料

- 沿用 lucide，但**导航必须带图标**（现在纯文字）。给每个模块定 1 个稳定图标。
- Logo：现有 `Radar` 图标可保留，做一个带"扫掠环"的动态版用于登录/加载，强化"开机/接入"的仪式感。

## 4. 布局与信息架构（应用外壳）

### 4.1 从"居中列" → "持久侧边栏 App Shell"

```
┌──────────────────────────────────────────────────────────────────────┐
│ ◐ Hatch Radar     ⌘K 搜索 / 跳转…        ◉采集 2m前 · 队列③ · W2     ☾  ◑Moss │ ← 顶栏：品牌 / 全局搜索 / 系统脉搏 / 主题 / 用户
├──────────────┬─────────────────────────────────────────────────────────┤
│ 工作区        │  洞察                                       [筛选] [导出]  │ ← 页面标题区（H1 24px + 副标题 + 右侧主操作）
│  ◈ 看板       │ ┌─────────────────────────────────────────────────────┐ │
│  ✦ 洞察   ●   │ │▌HIGH   r/startups · 2h        痛点 3 · 机会 2          │ │ ← 列表项：左侧强度色条
│  ▤ 帖子       │ │  "Why is invoicing still so painful for freelancers?" │ │
│              │ │  #billing #freelance #saas                            │ │
│ 运营          │ └─────────────────────────────────────────────────────┘ │
│  ⚡ 分析       │ ┌─────────────────────────────────────────────────────┐ │
│  ◷ 队列   ③   │ │▌MED    HackerNews · 5h        痛点 1 · 机会 3          │ │
│              │ └─────────────────────────────────────────────────────┘ │
│ 系统          │                                                          │
│  ⚙ 设置       │                                                          │
│  ⛉ 账户       │                                                          │
│  ▣ 审计       │                                                          │
│──────────────│                                                          │
│ ◑ Moss        │                                                          │
│  超级管理员    │                                                          │
└──────────────┴─────────────────────────────────────────────────────────┘
```

要点：

- 左侧栏固定 ~240px，可收起为图标条（窄屏自动收起为抽屉，复用已有 `sheet`/`sidebar` 组件）。
- 工作区**去掉全局 `max-w-5xl`**，改为**按页型设宽**：数据页（看板/队列/帖子/审计）全幅；阅读页（洞察详情/帖子详情）内容列收到 ~`max-w-3xl` 保证可读性。
- 顶栏从"导航栏"退化为"上下文栏"：面包屑/页面位置 + 全局搜索 + **系统脉搏** + 主题 + 用户菜单。导航职责整体下放到侧栏。

### 4.2 导航分组 = 权限分组（IA 的关键洞察）

把侧栏分三组，**正好对齐 RBAC 的三类能力**（数据浏览 / 运营操作 / 系统管理）：

| 分组       | 模块               | 权限键                                               |
| ---------- | ------------------ | ---------------------------------------------------- |
| **工作区** | 看板 / 洞察 / 帖子 | `insights:view` / `posts:view`                       |
| **运营**   | 分析 / 队列        | `analyze:run`                                        |
| **系统**   | 设置 / 账户 / 审计 | `settings:manage` / `accounts:manage` / `audit:view` |

好处：① 权限显隐天然落在分组上（某组全无权限就整组隐藏，比现在零散藏链接更干净）；② 用户对"我能干什么"形成清晰心智；③ 当前藏在右上用户菜单里的"账户管理/审计"（[user-menu.tsx:40](apps/web/src/components/user-menu.tsx#L40)）回归到"系统"组——它们是系统功能，不是"个人"功能。个人中心才留在用户菜单。

### 4.3 两个"成熟系统"标配（强烈建议）

**① 命令面板 ⌘K**（UI 库里已有 `command`/cmdk，几乎零依赖成本）
全局搜索 + 跳转 + 动作：跳到任意洞察/帖子、切换 active 模型、对选中帖发起分析、跳设置某 Tab、切主题。这是"专业感"密度最高的一个功能，且和"情报检索"的产品调性完美契合。

**② 系统脉搏（顶栏常驻）**
一个常驻的小指示器，复用现在已有的轮询（[site-nav.tsx:40](apps/web/src/components/site-nav.tsx#L40) 的队列红点 + dashboard 数据）：`◉ 采集 2分钟前 · 队列 3 · Worker 2在线`。`◉` 用 `--signal` 青色做缓慢呼吸动画。它让整个系统"看起来活着"，且把分散在看板/队列里的关键运行态提到全局可见。点击展开一个小浮层看详情。

### 4.4 响应式

- `lg` 以上：侧栏常驻；
- `md`：侧栏收成 64px 图标条；
- `<md`：侧栏变底部/抽屉，顶栏保留搜索与脉搏。
- 数据表格在窄屏走"关键列优先 + 次要列折叠进详情弹窗"（队列页已有详情弹窗模式，[queue.tsx:136](apps/web/src/pages/queue.tsx#L136)，可推广）。

## 5. 逐页重设计

> 格式：**现状 → 改成**。只列关键动作。

### 5.1 看板 → 指挥中心（Command Center）

- **现状**：KPI 卡 + 手搓 div 柱图 + 一堆 `BarRow`，视觉很将就，且只是"被动展示"。
- **改成**：
  - 顶部 KPI 行升级为**带趋势的指标卡**（数字 + 迷你 sparkline + 同比箭头），用 recharts（库已在）替手搓柱子。
  - 加一个**"信号流"主视觉**：近 14 天分析吞吐做成面积图，叠加抓取/失败标记——让人一眼看到"工厂在跑"。
  - Worker 卡片做成**带心跳脉冲**的状态卡（在线=青色呼吸，僵死=红色），CPU/内存用细环形进度。
  - 成本区做成**按模型堆叠条 + 总成本大数字（mono）**，这是老板最关心的，给它 C 位。
  - 把"待分析 N 篇"做成可点的行动入口 → 直达分析工作台。

### 5.2 洞察列表（核心价值页，重点打磨）

- **现状**：卡片信息密度中等，强度只是灰 badge，痛点摘要灰字一行。最重要的"信号强弱"被埋没。
- **改成**：
  - 列表项**左侧一条强度色条**（红/琥珀/绿），扫一眼就知道哪些是强信号——这是把"信号优先"原则落到最该落的地方。
  - 头部一行：`强度Pill` · `来源徽章（带平台色）` · 版块 · 相对时间；右侧 `痛点 N · 机会 N` 用 mono 对齐。
  - 标题做大（H3 15px→更突出），痛点摘要 2 行 clamp 保留。
  - 标签做成可点 Pill（已支持搜索跳转，保留）。
  - 顶部**工具栏**重做（见 6.4）：左筛选、右"导出批次"。加一个**密度切换**（舒适/紧凑）。
  - 加**强度分布的迷你直方图**在列表顶，作为"这批信号的概览"。

### 5.3 洞察详情（阅读页）

- **现状**：单卡片竖排，痛点有左色条（已不错），机会有灯泡图标。
- **改成**：
  - 收到阅读列宽（`max-w-3xl`），左信息/右目录的双栏感（大屏右侧浮"痛点/机会/研判"锚点导航）。
  - 痛点卡片强化：强度色条 + 证据 blockquote 用引号装饰 + 来源回链。
  - "人工研判（移动端同步）"做成**右侧信息卡**或顶部状态条，突出"这条已被谁判成什么"——呼应 web 是"闭环决策点"。
  - 顶部加**操作条**：查看原帖 / 站内帖子 / 复制链接 / （有权限时）重新分析。

### 5.4 帖子列表 / 详情

- 列表统一到 5.2 的"数据列表"语言（紧凑行：标题 + 来源 + 作者 + 时间 + 赞/评 + 分析状态徽章）。
- 详情：正文用更舒适的排版（不是现在的 `bg-muted` 灰块塞原文），评论树加层级引导线 + 折叠。

### 5.5 分析工作台

- **现状**：一个多选表格 + 模型下拉 + 运行按钮。
- **改成**：
  - 做成**"待办队列"工作台**：左侧待分析列表（多选），底部固定一条**批量操作条**（选中 N 篇 · 模型 [下拉] · `运行` ），选中态浮现，像收件箱批量操作。
  - 模型下拉显示**当前 active 模型 + Key 池健康**（active/cooling/invalid 小圆点），让人运行前就知道"现在用哪把枪、子弹够不够"。
  - 运行后给乐观反馈 + toast "已入队 N 篇 → 看队列"，并让顶栏脉搏队列数 +N。

### 5.6 队列（实时）

- **现状**：表格 + 状态筛选 Tab + 3s 轮询 + 行点开详情弹窗。结构其实不错。
- **改成**：
  - 状态筛选做成**带计数的 Segmented 控件**（已有计数，视觉升级）。
  - 行内"运行中"状态加**进度脉冲/转场**，失败行用 `--intensity-high` 左缘标记。
  - 顶部加一条**迷你时间线**（近 1 小时入队/完成节奏），强化实时感。
  - 详情弹窗保留，补"重试/取消"动作（若后端支持）。

### 5.7 设置（Tabs：模型 / 来源 / 运行参数）

- **现状**：Tabs + 三个管理器组件，功能全但视觉平。
- **改成**：
  - 左侧设置子导航（替代顶部 Tabs，设置项多时更稳），右侧内容。
  - **模型与密钥**：每个 provider 一张卡，Key 池做成小表：`标签 · sk-a…wxyz · 优先级 · ●active/◐cooling(7m)/✕invalid`，状态用语义色点 + 行内"测试/复位/停用"。突出"哪个是 active 模型"。
  - **数据来源**：连接器卡（带"上次测试通过 2h前 ✅"）+ 爬虫计划勾选表；Reddit 无可用连接器时**整组置灰 + Tooltip 说明**（呼应三道门禁的前端闸）。
  - 危险/敏感操作（删 Key、改 base_url 强制重填）统一用 `alert-dialog` 二次确认 + 文案预警。

### 5.8 账户 / 权限 / 审计

- **账户管理**：用户列表（表格）+ 右侧 `Sheet` 编辑（资料/角色/权限/设备）。权限编辑**按三组勾选**（数据/运营/系统），顶部给"研判员/只读/自定义"**预设快选**（呼应 RBAC 附录的默认预设）。设备区列出设备 + 最近活跃 + 到期倒计时 + 强踢；"赋予设备"走激活码 + 二维码弹窗。
- **审计**：表格保留，`action` 用语义色 Pill 分类（登录/账户/密钥/分析/导出），加时间范围 + 操作者筛选 + 导出。
- **个人中心**：保留 Tabs（资料/安全/我的权限），"活跃会话"卡片化、当前会话高亮、一键登出其他。

### 5.9 登录页（第一印象，仪式感）

- **现状**：居中小卡片，朴素表单。
- **改成**：左侧品牌区（深色 + 缓慢雷达扫掠动画 + 一句产品主张"把噪音炼成信号"），右侧登录卡。强制改密用同一视觉语言。这是"开机/接入指挥台"的第一印象，值得花一屏。

## 7. 让它"活着、成熟"的微交互

这些是"廉价 vs 成熟"的真正分水岭，逐条都低成本：

1. **乐观更新**：分析入队、勾选、改设置即时反馈，不等请求回来才动。
2. **实时不闪烁**：已用 `keepPreviousData`，再加"新数据轻微高亮一下"而不是整屏重渲。
3. **键盘流**：⌘K、列表 `j/k` 上下、`Enter` 进详情、`Esc` 关弹窗。专业工具的肌肉记忆。
4. **过渡**：路由切换、弹窗、列表增删都有 120–200ms 过渡。
5. **脉冲语言**：实时/在线状态统一用 `--signal` 青色呼吸，全站一致 = 一眼认得"这是活的"。
6. **可达性**：聚焦环（已有 ring）保留并用品牌色；语义色都配文字/图标，不只靠颜色传达（强度同时有色 + 文字 HIGH/MED/LOW）。

## 8. 落地路线（按"性价比/风险"排序，可分批合并）

因为是 shadcn + Tailwind v4 CSS-first，大头是**改令牌 + 加外壳**，不是重写业务。

- **P0 · 令牌换气质（半天～1天，零组件改动，收益最大）**
  改 [globals.css](packages/ui/src/styles/globals.css)：品牌色 + 信号色 + 强度色 token + 中性色上色 + 海拔阴影 + 字阶/字体。**这一步做完，全站气质立变**，且可独立验证、随时回退。

- **P1 · 应用外壳（1～2天，结构性收益）**
  新增侧栏布局替换 [layout.tsx](apps/web/src/layout.tsx) 的居中列；导航分组对齐权限；顶栏改"上下文栏" + 系统脉搏；去全局 max-width、按页型设宽。

- **P2 · 核心页打磨（洞察列表/详情 + 看板）**
  强度色条、KPI/图表升级、指挥中心化。这是用户停留最久、价值最高的页。

- **P3 · 列表统一 + 工具栏 + 组件升级**
  统一数据列表语言、Toolbar、空/载/错、Toast。

- **P4 · ⌘K 命令面板 + 微交互 + 登录页仪式感**
  画龙点睛，把"专业感"拉满。

每个 P 都能独立交付、独立验证、不破坏后端契约。

## 附：与产品需求的一致性校验

- ✅ **权限驱动 UI**：导航分组 = 权限分组，组级显隐更干净；服务端 `@RequirePermission` 仍是权威，前端只管体验。

- ✅ **闭环决策点定位**：洞察详情突出"人工研判"、分析工作台像"批量收件箱"，强化 web 作为"人工介入唯一场所"的角色。
- ✅ **实时数据工厂**：系统脉搏 + 指挥中心把"抓取/队列/worker/成本"提到一眼可见，呼应后端的实时性。
- ✅ **安全细节**：Key 脱敏 + 状态色点、改 base_url 重填、删 Key 二次确认、设备到期倒计时——都在视觉层得到强化而非削弱。
